# Sperm chromosome genotyping and recombination: source-code logic

Source-code location in this package:

`source_code/genotyping_and_recombination/`

This directory contains a compact, runnable reference implementation for the manuscript Methods sections **Sperm chromosome genotyping** and **Recombination**. It is designed for transparent peer-review testing with the supplied demo data while preserving the same core logic used by the full-scale workflow.

## Workflow overview

```text
chromosome haplotype definitions
        +
sperm SNP genotype/depth table
        +
chromosome sizes
        |
        v
SNP filtering and SNP-to-haplotype assignment
        |
        v
10-kb window haplotype calling
        |
        v
chromosome-level haplotype composition summary
        |
        v
local SNP haplotype correction with flanking SNPs
        |
        v
haplotype-switch recombination calls
        +
adjacent inconsistent-SNP gene-conversion candidates
```

## Input files

The demo inputs are in:

`demo_data/genotyping_and_recombination/`

### `chromosome_haplotypes.tsv`

Defines informative haplotype bases for chromosome-level genotyping. The expected fields are:

- `chrom`: chromosome identifier.
- `pos`: 1-based SNP position.
- one or more haplotype columns, for example `hap1`, `hap2`, `hap3`.

This table represents phased chromosome haplotypes generated by upstream HWM analysis/phasing workflows.

### `demo_sperm_snps.tsv`

Defines SNP-level observations for a sperm sample. The expected fields are:

- `sample`: sample identifier.
- `chrom`: chromosome identifier.
- `pos`: 1-based SNP position.
- `base`: observed sperm base.
- `depth`: read depth supporting the site.

### `chrom_sizes.tsv`

Defines chromosome lengths used to create fixed-size windows.

- `chrom`: chromosome identifier.
- `length`: chromosome length.

## Script 1: `genotype_sperm_chromosomes.py`

Purpose: perform SNP filtering, assign SNPs to chromosome haplotypes, summarize haplotypes in fixed windows, and produce chromosome-level calls.

### Main filtering logic

The script applies the reader-facing implementation of the Methods rules:

1. Remove non-informative haplotype sites where chromosome haplotypes have the same genotype.
2. Remove haplotype sites containing `N`.
3. Remove sperm SNP sites with missing bases.
4. Remove SNP sites supported by fewer than `--min-depth` reads; demo/default Methods value: 6.
5. Remove SNP sites with depth greater than `--max-depth`; demo/default Methods value: 30.
6. Remove SNP sites closer than `--min-snp-distance` bp to adjacent retained SNPs; demo/default Methods value: 5 bp.
7. Remove isolated SNPs whose haplotype assignment conflicts with both immediate neighboring assignments on the same chromosome.

### SNP-to-haplotype assignment

For each retained SNP:

1. The observed sperm base is compared with bases in the chromosome haplotype definition table.
2. If the observed base uniquely matches one haplotype, that haplotype is assigned.
3. If multiple haplotypes share the observed base, the assignment is retained as a multi-haplotype state when informative for downstream logic.
4. If no haplotype matches, the SNP is excluded or marked as unassigned depending on context.

### Window-level haplotype calling

The script partitions each chromosome into fixed windows using `--window-size`.

For each sample/chromosome/window:

1. Count retained informative SNPs in the window.
2. If the count is below `--min-window-snps`, label the window as insufficient.
3. Otherwise, calculate haplotype support counts.
4. Assign the window to the best-supported haplotype or haplotype combination.

Methods-aligned demo values:

- `--window-size 10000`
- `--min-window-snps 10`

### Output files

Written to the selected genotyping output directory, for example:

`demo_output/genotyping_and_recombination/genotyping/`

- `filtered_snp_haplotypes.tsv`: retained SNPs and assigned haplotypes.
- `removed_snps.tsv`: excluded SNPs and exclusion reasons.
- `window_haplotypes.tsv`: window-level haplotype calls.
- `chromosome_calls.tsv`: chromosome-level haplotype composition summaries.

## Script 2: `call_recombination.py`

Purpose: identify haplotype-switch recombination events and adjacent inconsistent SNP clusters representing candidate gene-conversion regions.

### Input logic

The script uses:

- `window_haplotypes.tsv` from `genotype_sperm_chromosomes.py`.
- `filtered_snp_haplotypes.tsv` from `genotype_sperm_chromosomes.py`.

### Local SNP haplotype correction

For each SNP, the script inspects neighboring haplotype context using `--flank-snps`, corresponding to the Methods rule of considering upstream and downstream SNPs.

Demo/Methods-aligned value:

- `--flank-snps 20`

Correction logic:

1. Collect upstream and downstream informative SNP haplotypes on the same chromosome.
2. If the flanking context supports a consistent haplotype and the focal SNP is discordant, correct the focal SNP to the locally supported state.
3. If the focal site remains ambiguous after local context evaluation, exclude it from recombination/gene-conversion calling.

### Recombination-event calling

Window-level recombination events are called when adjacent informative windows on the same chromosome show a haplotype state change.

Output:

- `recombination_events.tsv`

Typical fields include:

- sample;
- chromosome;
- left and right window coordinates;
- left and right haplotype states;
- event type.

### Candidate gene-conversion region calling

The script also scans corrected SNP haplotypes to identify local clusters inconsistent with the broader chromosome/window haplotype state.

Rules represented in the reference implementation:

1. Identify SNPs whose corrected haplotype differs from the expected local chromosome/window haplotype.
2. Group adjacent inconsistent SNPs.
3. Report a candidate region when a group contains two or more adjacent inconsistent SNPs.

Output:

- `gene_conversion_regions.tsv`

Typical fields include:

- sample;
- chromosome;
- start and end position;
- number of supporting SNPs;
- observed haplotype states;
- expected haplotype state.

## Demo commands

Run from the package root:

```bash
python3 source_code/genotyping_and_recombination/genotype_sperm_chromosomes.py \
  --haplotypes demo_data/genotyping_and_recombination/chromosome_haplotypes.tsv \
  --snp-genotypes demo_data/genotyping_and_recombination/demo_sperm_snps.tsv \
  --chrom-sizes demo_data/genotyping_and_recombination/chrom_sizes.tsv \
  --window-size 10000 \
  --min-window-snps 10 \
  --min-depth 6 \
  --max-depth 30 \
  --min-snp-distance 5 \
  --outdir demo_output/genotyping_and_recombination/genotyping

python3 source_code/genotyping_and_recombination/call_recombination.py \
  --window-genotypes demo_output/genotyping_and_recombination/genotyping/window_haplotypes.tsv \
  --snp-haplotypes demo_output/genotyping_and_recombination/genotyping/filtered_snp_haplotypes.tsv \
  --flank-snps 20 \
  --outdir demo_output/genotyping_and_recombination/recombination
```

## Demo outputs

Expected output directory:

`demo_output/genotyping_and_recombination/`

The supplied demo output contains:

- four genotyping output tables;
- three recombination/gene-conversion output tables;
- one haplotype-switch recombination event;
- one candidate gene-conversion region.

## Relationship to the full workflow

The full manuscript workflow uses upstream HWM analysis outputs, including mapped reads, filtered SNPs, phased haplotypes, chromosome-depth summaries and haplotype assignment tables. This compact module provides a transparent implementation of the final genotyping and recombination logic, while `source_code/HWM.backScipt/` contains the broader production-oriented upstream workflow.
