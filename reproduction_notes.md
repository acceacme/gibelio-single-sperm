# Reproduction notes

This document maps the manuscript data-processing workflows to the code modules included in this package.

## 1. HWM analysis workflow

Code module: `source_code/HWM.backScipt/`

This module represents the production-oriented upstream and intermediate analysis workflow used for single-sperm data processing.

Typical inputs:

- raw short-read FASTQ files;
- reference genome(s) and chromosome-size files;
- sample lists and per-chromosome interval files;
- BAM/VCF/depth intermediates generated by upstream workflow steps.

Typical outputs:

- read-quality summaries;
- sorted/processed/indexed BAM files;
- chromosome-depth and coverage matrices;
- depth-ratio chromosome-content summaries;
- SNP and filtered VCF files;
- distance/PCA/phasing outputs;
- haplotype-assignment and genotype-format tables.

## 2. Sperm chromosome genotyping

Code module: `source_code/genotyping_and_recombination/`

Inputs:

- phased chromosome haplotypes;
- SNP genotype/depth tables for each single-sperm sample;
- chromosome-size table.

Outputs:

- filtered SNP table;
- 10-kb window haplotype table;
- chromosome-level genotype/phasing calls.

Algorithmic parameters represented in the demo/reference implementation:

- minimum support for non-missing SNPs: 6 reads;
- maximum accepted SNP depth: 30x;
- minimum distance between adjacent SNPs: 5 bp;
- window size: 10 kb;
- minimum informative SNPs per window: 10.

## 3. Recombination and candidate gene-conversion detection

Code module: `source_code/genotyping_and_recombination/`

Inputs:

- cleaned SNP haplotypes;
- window-level haplotypes;
- chromosome genotype calls.

Outputs:

- recombination switch table;
- candidate gene-conversion regions;
- corrected SNP haplotype table.

Algorithmic parameters represented in the demo/reference implementation:

- samples with excessive missing SNPs are excluded in the full workflow;
- SNP haplotypes are corrected using 20 upstream and 20 downstream SNPs;
- SNPs with multiple possible haplotypes after correction are excluded;
- two or more adjacent inconsistent SNPs are grouped into candidate gene-conversion regions.

## 4. Gonad-enriched expression analysis

Code module: `source_code/gonad_enriched/`

Typical inputs:

- multi-tissue RNA-seq expression matrix;
- gene locations/chromosome assignments;
- chromosome-frequency table from single-sperm analysis.

Typical outputs:

- merged tissue-level expression matrix;
- testis- and ovary-enriched gene sets;
- chromosome-level enriched-gene expression metrics;
- optional correlation and visualization outputs.

## 5. Broader manuscript-associated analyses

The broader analysis framework also includes breakpoint-proximal gene extraction, functional enrichment, observed-vs-random genomic distance comparisons, regional heterozygosity/repeat-density summaries, chromosome presence/absence enrichment summaries, paternal/sex-specific sequence analyses, whole-genome collinearity and putative centromere-position inference.

These broader analyses require the original large sequencing datasets, reference assemblies, gene/repeat annotations, centromere tables and production intermediate files. The package demos are designed to validate representative software logic and file formats; they are not intended to regenerate every full-scale result from raw sequencing data.
